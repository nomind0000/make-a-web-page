Lesson One

What is the web?

The web is a set of hypertext markup language (html) files, the servers upon which they reside, and the network of individual computers to which the servers connect using the http protocol. The pages are identified by URLs and are joined by hyperlinks.


What is html?

HTML is the coding language that creates, collects, and formats the content of pages on the web. It produces text content, "markup," i.e., what it looks like, and hyperlinks to other web pages.


Can you explain tags, elements, and attributes?

It seems that tags are super important, controlling formatting, importing content, and identifying textual content. A tag is enclosed in angled brackets ("less than" and "greater than")that surround it. An element comprises an opening tag, a closing tag, and the content between them. An attribute is a property of a tag that has a value. Consider, for example, the expression below:


<a href="codepen.io/NoMind/pen/VYJRVK"> NoMind's first web page is here!</a>


The expression as a whole is an element with an opening anchor tag, which includes the href attribute that specifies a hyperlink; the content, that is, the expression that appears to the reader as a hyperlink; and a closing anchor tag, </a> . This is what the reader sees:


NoMind's first web page is here!


You can insert a photo of NoMind using html. The <img> tag, referred to as a "VOID" tag because it has an opening but no closing tag, makes this easy to do. Using the src (for source) attribute, which has the value of the URL where the photo can be found, brings the photo to your page.

Where were we? Oh, yes--TAGS. Of tags there are two important species. One is referred to as inline and operates on a single line of code within which it is embedded. The other species is a block, which defines a rectangular box around the content.

Inline tags such as <b>,<span>,and <img>seem to be more common, at least at stage zero, than block tags such as <p> and <div>. The notion of a defined block is easily understood by visualizing a box around a paragraph of text defined by <p>.


How do <span> and <div> work?

Block tags "contain" things. This means, I think, that you can manipulate the contents of the tag in uniform ways. Thus, it is possible to create uniform style features for an element. Assigning class attributes allows you to style the content of multiple tags in the same way. <div> creates a box within which all features assigned to that element are uniform, unless a nested box requires something different. Similarly, different elements designating the same class, as <div class="class"> or <h2 class="class"> can be treated the same way as specified by the user in CSS, the companion program to HTML that supplies most characteristics of style.

The "div" tag contains things in blocks, whereas the "span" tag contains things "inline," i.e., within a sequence of text characters which would not necessarily lend themselves to treatment within a rectangular block, as when a sentence begins in one line and ends in another. These tags use the same format for assigning classes, for example:


<span class="foo">
text_formatted_as_foo
</span>

DOM and CSS

HTML defines elements through tags, uses attributes with classes, and, thanks to "DOM," turns tags into elements in a tree."DOM" is the acronym for "Document Object Model," a cross-platform and language independent convention for representing and interacting with HTML objects. The expression, "representing and interacting with HTML objects" sounds like gobbledigook at this point. But DOM seems to be behind it all, somewhere that we cannot see quite yet. "CSS," for Cascading Style Sheets, is the language that provides HTML objects with font, color, background, position, and other attributes. An HTML document includes information on CSS styles, and this information can be seen with our Chrome browser in the style panel on the right. Most significantly, the puzzle is easy to put together because everything on the page is contained in a box (rectangle). This is demonstrated on a posterboard and through an exercise in cutting rectangular shapes from a printed mock-up of a page. In a simple example, four boxes, two of which are nested within a third, are defined by the div tag. They arrange a title, of sorts above an image and associated text, thus:

<div>text</div>

</div>


From this code we can add "classes," though for now the function of these remains something of a mystery. For example:

Lesson Three

CSS: Friend or Foe?

While the intent of CSS, or Cascading Style Sheets, is to simplify style coding by grouping stylistic characteristics into classes that need be defined only once, simplicity does not stand out as its most immediate quality. Actually, I do understand. CSS works through a cascading hierarchy of commands in which many style commands are inherited from parent to child, or from more general to more specific commands. The wikipedia article on inheritance helps to explain this idea and its application:


"Descendant elements may inherit CSS property values from any ancestor element enclosing them. In general, descendant elements inherit text- related properties, but box-related properties are not inherited."


I must confess that I am not quite sure what this distinction means, but when I figure it out, this note should be a helpful reminder of something important. In every case (I think), the command closest to the displayed content, that is, the most "specific" command, trumps the higher level command. The hierarchy is created by nesting blocks within blocks. For our purposes, the most important blocks are supplied by <p> and <!DOCTYPE>, <html>, <body, and other elements that appear in hierarchies of style commands. We also learn that there are three main ways to code style, as follows:

  1. The style can be placed in the head. This is fine for short documents where a particular style applies throughtout the document.
  2. The style can be placed in a separate *.css file that is linked to the html document through a link command. This method provides consistency, avoids repetition, and simplifies editing. It is usually the preferred method.
  3. The <style>makes this possible, but it is greatly discouraged for the same reasons that method two is preferred. Too much of this and I bet your project will be sent back. Unfortunately, I already used this command before getting to lesson 3.

Colors, which can be defined in more than one way, are applied to containers using these methods. The instructors notes suggest consulting the linked reference. Colors and other style properties can be assigned values that apply to a certain kind of element or to a certain kind of user-defined class.


Selection by Element or Class

"Selection" refers to the assignment of a style trait either to an element or to a class. An element to which such an assignment is made is called a selector. Thus, for example, the following CSS code assigns background and font colors to the h1 element:

h1 {

background-color : black;
color: white;

}


This assigns a block background and a white text color to every h1 element.

Classes provide greater flexibility assigning a particular property value to any element the user wishes. The class is easily assigned as follows: <ElementName class="AssignedClassName"> . The property is called from the CSS file much as it would be for an element. The class is distinguished from the element in CSS by an introductory period, thus:

.code {

font-family: Courier;

}


In this way any element given the class called "code" will cause the text for that element to assume Courier font.
